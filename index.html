VPS Armor

Because your VPS deserves better than "I'll secure it later"

One command. Five layers of protection.

You spun up a VPS for your cool self-hosted project. Awesome! But did you remember to update it? Enable a firewall? No? We've all been there.

VPS Armor is a simple script that handles the boring security stuff so you can get back to the fun part.

üêß Works on Ubuntu LTS (22.04, 24.04) and Debian Stable (Bookworm, Trixie)

Run this as root on your VPS:

bash
curl -sSL https://vpsarmor.com/armor.sh | bash

Not running as root? Prefix with sudo

What does it do?

üîç

OS Check

Verifies you're on a supported Ubuntu LTS or Debian Stable release. No surprises on weird distros.

üì¶

Full Update

Runs apt update and apt dist-upgrade to patch all the things.

ü§ñ

Unattended Upgrades

Installs unattended-upgrades and uses distro defaults for automatic security updates.

üö´

Fail2Ban

Bans IPs that try to brute-force your SSH. Goodbye, script kiddies.

üß±

UFW Firewall

Ensures SSH/HTTP/HTTPS are allowed and preserves any existing UFW rules.

The script (armor.sh)

armor.sh 
#!/bin/bash
# VPS Armor - Basic security hardening for Ubuntu LTS and Debian Stable
# https://vpsarmor.com
# License: BSD 3-Clause

set -euo pipefail

echo "üõ°Ô∏è  VPS Armor - Hardening your server..."
echo ""

# Colors for output
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# Check if running as root
if [[ $EUID -ne 0 ]]; then
   echo -e "${RED}Error: Please run as root (or use sudo)${NC}"
   exit 1
fi

# Load OS information
if [[ ! -f /etc/os-release ]]; then
    echo -e "${RED}Error: Cannot detect OS. /etc/os-release not found.${NC}"
    exit 1
fi
. /etc/os-release

# Check for supported OS
echo "üîç Checking OS compatibility..."
if [[ "$ID" == "ubuntu" ]]; then
    case "$VERSION_ID" in
        22.04|24.04)
            echo -e "${GREEN}‚úì Ubuntu $VERSION_ID LTS detected${NC}"
            ;;
        *)
            echo -e "${RED}Error: Ubuntu $VERSION_ID is not an LTS release.${NC}"
            echo "Supported: 22.04, 24.04"
            exit 1
            ;;
    esac
elif [[ "$ID" == "debian" ]]; then
    case "$VERSION_CODENAME" in
        bookworm|trixie)
            echo -e "${GREEN}‚úì Debian $VERSION_CODENAME detected${NC}"
            ;;
        *)
            echo -e "${RED}Error: Debian $VERSION_CODENAME is not a stable release.${NC}"
            echo "Supported: bookworm (12), trixie (13)"
            exit 1
            ;;
    esac
else
    echo -e "${RED}Error: Unsupported OS '$ID'.${NC}"
    echo "VPS Armor only supports Ubuntu LTS and Debian Stable."
    exit 1
fi

# Update system
echo ""
echo "üì¶ Updating system packages..."
apt-get update -qq
DEBIAN_FRONTEND=noninteractive apt-get dist-upgrade -y -qq

# Install security packages
echo ""
echo "üîß Installing security tools..."
DEBIAN_FRONTEND=noninteractive apt-get install -y -qq \
    unattended-upgrades \
    apt-listchanges \
    fail2ban \
    ufw

# Configure fail2ban
echo ""
echo "üö´ Configuring fail2ban..."
systemctl enable fail2ban --quiet
systemctl start fail2ban

# Configure UFW
echo ""
echo "üß± Setting up firewall (UFW)..."
if ufw status | grep -q "^Status: active"; then
    ufw allow 22/tcp comment 'SSH' > /dev/null
    ufw allow 80/tcp comment 'HTTP' > /dev/null
    ufw allow 443/tcp comment 'HTTPS' > /dev/null
    echo -e "${GREEN}‚úì UFW already active. Existing rules preserved.${NC}"
else
    ufw default deny incoming > /dev/null
    ufw default allow outgoing > /dev/null
    ufw allow 22/tcp comment 'SSH' > /dev/null
    ufw allow 80/tcp comment 'HTTP' > /dev/null
    ufw allow 443/tcp comment 'HTTPS' > /dev/null
    ufw --force enable > /dev/null
    echo -e "${GREEN}‚úì Firewall enabled. Allowed ports: 22, 80, 443${NC}"
fi

# Summary
echo ""
echo "=========================================="
echo -e "${GREEN}üõ°Ô∏è  VPS Armor complete!${NC}"
echo "=========================================="
echo ""
echo "Your server now has:"
echo "  ‚úì All packages updated"
echo "  ‚úì Unattended upgrades installed (distro defaults)"
echo "  ‚úì Fail2ban protecting SSH"
echo "  ‚úì UFW firewall active (existing rules preserved)"
echo ""
echo -e "${YELLOW}Note: If you need additional ports, run:${NC}"
echo "  ufw allow <port>/tcp"
echo ""
echo "Stay safe out there! üöÄ"

‚ö†Ô∏è Heads up

  • By default, UFW will block other ports. If you're running services on other ports (like 3000 for Node.js) and you need to access them directly from outside (not through a reverse proxy), open them with: ufw allow 3000/tcp
  • Test on a fresh VPS first. We've tested this, but your setup might be... unique.
  • This is baseline security. It's a great start, not a complete solution. Consider SSH keys, Docker security, etc.
